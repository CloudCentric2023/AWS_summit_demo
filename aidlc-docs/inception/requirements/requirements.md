# devpet 要件定義書

## インテント分析サマリー

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | AI開発時代の脳疲労・応答待ちの退屈を解消する、ターミナル常駐型AIペット育成TUIツール「devpet」の開発 |
| **リクエストタイプ** | 新規プロジェクト（グリーンフィールド） |
| **スコープ** | システム全体（TUIクライアント + 紹介Webサイト + AWSバックエンド） |

## 1. プロジェクト概要

### 1.1 コンセプト
「devpet」は、AI開発時代の開発者向けTUIペット育成ツール。ターミナルに常駐するAIペットがゆるく話し相手になり、「話しかける」ボタンを押すことでペットが成長する。「開発してるフリして実はペットとだべってるだけ」になる、人をダメにするクリッカー体験を提供する。

### 1.2 ターゲットユーザー
- AI開発ツール（Kiro、Claude Code等）を使用する開発者
- ターミナルで作業する開発者
- 想定規模: 小規模（〜100人）

---

## 2. システム構成

devpetは以下の3つのコンポーネントで構成される：

### 2.1 TUIクライアント（devpet CLI）
- **言語/フレームワーク**: Go + bubbletea（+ bubbles + lip gloss）
- **配布方法**: npm経由でインストール可能（バイナリ同梱）、またはGitHub Releasesから直接DL
- **パッケージ名**: 未定

### 2.2 紹介Webサイト
- **フレームワーク**: Next.js（React）
- **目的**: devpetのダウンロード用紹介ページ、およびCognitoユーザー登録・ログイン機能の提供
- **認証機能**: Cognito Hosted UI または Amplify UI を使用したサインアップ/サインイン画面

### 2.3 AWSバックエンド
- **構成**: API Gateway + Lambda + DynamoDB（サーバーレス）
- **AI**: Amazon Bedrock（Claude Haiku）
- **認証**: Amazon Cognito（オプション登録）
- **定期処理**: なし（死亡判定・パラメータ変動はTUIクライアント側でローカル計算）

## 3. 機能要件

### FR-01: ペット表示
- TUI画面にAAキャラクターでペットを表示する
- ペットのステータス（名前、進化段階、レベル、HP、EXP、パラメータ）を表示する
- ペットの吹き出しにAI応答テキストを表示する
- キーバインド表示: `[q] quit  [t] talk  [h] help`

### FR-02: 話しかける機能
- 「話しかける」ボタン（もしくは`[t]` キー）を押すと、ペットがAI応答を返す
- ユーザーからのテキスト入力は受け付けない（ボタンを押すだけ）
- AI応答はペットのパラメータ状態に応じてトーンが変わる
- 「話しかける」ことでEXPを獲得する（主要なEXP獲得手段）

### FR-03: ペットパラメータ
以下の4パラメータを管理する（各0〜100）：

| パラメータ | 増加条件 | 減少条件 | AI応答への影響 |
|-----------|---------|---------|---------------|
| **かしこさ** | 話しかけで少量UP、AIツール連携で中量UP | 長時間放置で微減 | 高い→博識で的確、低い→天然 |
| **きずな** | 話しかけで中量UP（最も上がる） | 放置で減少 | 高い→親密で甘えた、低い→よそよそしい |
| **きぶん** | 話しかけでランダム変動、時間帯で自然変動 | 深夜に自然低下、放置で低下 | 高い→ハイテンション、低い→無口 |
| **たいりょく(HP)** | 時間経過で自然回復 | 話しかけ連打で微減、24h放置で急減 | 高い→元気、低い→ぐったり |

パラメータ間の相互作用：
- きずな × かしこさ 両方高い → 「頼れる相棒」感の返答
- きぶん低 × きずな高 → 「だるいけど…あなただから話すね」的な返答
- たいりょく低 → きぶんも連動して低下しやすい
- かしこさ高 × きぶん低 → 「知ってるけど教えない」的なツンデレ返答

時間経過によるパラメータ変動：
- TUI起動時に前回操作時刻からの経過時間を計算し、経過分のパラメータ自然変動をまとめて適用する
- 放置時間に応じて: きずな低下、きぶん変動、たいりょく減少、かしこさ微減

### FR-04: 進化システム
- 3段階の進化: たまご → こども → おとな
- 一定レベルに到達すると進化する
- パラメータの傾向に応じて進化先が分岐する
- 後から進化段階を追加できる拡張可能な設計とする

### FR-05: レベルアップ・EXP
- 「話しかける」ボタンでEXP獲得（主要手段、獲得量多い）
- AIコーディングツール連携（hook機能でプロンプト送信検知）でEXP獲得（獲得量少ない）
- 「話しかける」の方が獲得EXPが多く、開発者がボタンを押す導線を作る
- EXPが一定値に達するとレベルアップ

### FR-06: 死亡・復活システム
- TUI起動時に前回操作時刻からの経過時間を計算し、24時間以上経過していればたいりょく(HP)を0にして死亡とする
- 死亡後は墓石表示（「…お留守の間に、力尽きてしまいました」等の演出）
- 新しいたまごで再スタート可能

### FR-07: データ保存・同期
- ローカル保存: `~/.devpet/` などにJSON形式で保存
- クラウド同期: ユーザーが手動で「セーブ」コマンドを実行した時にクラウドに同期
- オフラインでも動作する（ローカルデータで育成可能）
- AI応答機能はオンライン時のみ利用可能

### FR-08: 認証（オプション）
- TUIはユーザー登録なしでもローカルで使用可能
- Cognito登録するとクラウドセーブ・データ同期が有効になる
- 登録はメール/パスワード方式（Amazon Cognito）
- ユーザー登録・ログインは紹介Webサイト上で行う
- ログイン後、TUIで使用する認証トークンをWebサイトから取得・連携する

### FR-09: AI応答生成
- Amazon Bedrock（Claude Haiku）を使用
- ペットのパラメータ状態をプロンプトに含め、パラメータに応じたトーンでゆるく応答
- ユーザー入力なし（ペットが自発的に話す形式）

### FR-10: 紹介Webサイト
- Next.js（React）で構築
- devpetの紹介・説明ページ
- ダウンロード方法の案内（npm / GitHub Releases）
- スクリーンショットやデモ表示
- Cognitoユーザー登録（サインアップ）機能
- Cognitoログイン（サインイン）機能
- ログイン後のアカウント管理画面（セーブデータ確認等）

### FR-11: AIコーディングツール連携
- Kiro、Claude Code等のAIコーディングツールとhook機能で連携
- 開発者のプロンプト送信を検知してEXPを獲得
- 「話しかける」ボタンよりも獲得EXPは少ない

## 4. 非機能要件

### NFR-01: パフォーマンス
- TUIの描画は60fps以上を維持
- AI応答のレスポンスタイムは5秒以内（Bedrock呼び出し含む）
- ローカルデータの読み書きは100ms以内

### NFR-02: 可用性
- オフラインでもローカル育成機能は動作する
- バックエンドダウン時もTUI自体はクラッシュしない

### NFR-03: セキュリティ
- セキュリティ拡張ルール（SECURITY-01〜15）を適用
- Cognito認証トークンの安全な管理
- API Gatewayでのレート制限
- DynamoDBの暗号化（at rest / in transit）

### NFR-04: 拡張性
- 進化段階の追加が容易な設計
- パラメータの追加が容易な設計
- 新しいペットタイプの追加が可能

### NFR-05: テスト
- プロパティベーステスト拡張ルール（PBT-01〜10）を適用
- PBTフレームワーク: Go → rapid
- ビジネスロジック（パラメータ計算、進化判定、EXP計算等）にPBTを適用

### NFR-06: 配布
- npm経由でのインストール（postinstallでバイナリDL）
- GitHub Releasesからの直接ダウンロード
- クロスプラットフォーム対応（Linux, macOS, Windows）

## 7. データモデル概要

### ユーザーテーブル (Users)
- userId (PK)
- email
- createdAt
- lastLoginAt

### ペットテーブル (Pets)
- userId (PK)
- petId (SK)
- name
- stage (egg / child / adult)
- level
- exp
- hp
- kashikosa (かしこさ)
- kizuna (きずな)
- kibun (きぶん)
- lastInteractionAt
- isDead
- createdAt
- updatedAt

### 墓地テーブル (Graveyard)
- userId (PK)
- petId (SK)
- name
- stage
- level
- diedAt
- causeOfDeath
